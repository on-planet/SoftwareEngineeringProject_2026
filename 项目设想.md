# QuantPulse 产品功能文档

> 本文档描述 QuantPulse 平台当前已上线的全部功能，面向产品演示、答辩汇报和用户介绍。

## 1. 项目定位

QuantPulse 是一个面向 A 股、港股、美股场景的 **T-1 日终金融数据分析平台**，提供市场行情、基本面分析、新闻舆情、宏观联动、组合诊断和策略回测等能力。

**核心特点**：
- 数据结构化、可追溯、可扩展
- 前端请求级缓存 + 后端 Redis/内存双层缓存
- 完整的管理员后台 + 访问分析可视化
- 支持 Docker 一键部署和 CI/CD 自动发布

## 2. 技术栈（实际采用）

| 层级 | 技术 |
|------|------|
| ETL | Python 3.12 + AkShare + Baostock + pysnowball |
| API | FastAPI + SQLAlchemy + Pydantic |
| 前端 | Next.js 13 + React 18 + TypeScript + ECharts |
| 数据库 | PostgreSQL 15 |
| 缓存 | Redis 7 + 内存缓存 |
| 部署 | Docker Compose + GitHub Actions CI/CD |

## 3. 功能模块（已上线）

### 3.1 指数看板

- 展示 A 股主要指数（上证、深证、创业板）
- 展示港股主要指数（恒生指数、国企指数、恒生科技）
- 日终数据，ETL 每日自动更新
- 首页卡片式布局，支持数据快速概览

### 3.2 股票列表与个股详情

- **股票列表**：按市场（A 股、港股）分页展示，支持搜索
- **个股详情**：
  - 基础信息（名称、行业、市场）
  - K 线图与日线数据
  - 基本面评分与财务指标
  - 新闻列表与情绪标签
  - 相关事件（回购、增减持、重大公告）
- 首屏/二屏拆分加载，优化首屏渲染速度

### 3.3 基本面评分

- 盈利质量维度（ROE、现金流/净利润）
- 成长性维度（营收增速）
- 风险维度（负债率、短债现金比）
- 汇总为 0-100 的综合评分
- 生成一句话诊断总结

### 3.4 新闻与事件

- 个股页展示相关新闻列表
- 新闻情绪标签（正面 / 中性 / 负面）
- **新闻关系图**：在洞察页展示新闻传播链和影响链
- 事件统计：回购、增减持、财报日历、重大公告

### 3.5 宏观数据

- 国内宏观：CPI、PPI、PMI、利率等
- 国际宏观：人民币汇率、美债收益率、原油、黄金
- 宏观快照与历史序列图表展示

### 3.6 期货数据

- 展示期货品种日终行情
- 支持周度数据查看

### 3.7 个人空间（Stats）

- **观察标的**：用户自选股票列表，支持批量导入
- **已买标的**：用户持仓录入，记录成本、数量、买入日期
- **组合诊断报告**：
  - 持仓总览（成本、市值、盈亏）
  - 行业暴露与市场分布
  - Top 持仓、风格标签
- **组合压力测试**：
  - 内置情景（市场暴跌、行业危机等）
  - 自定义规则压力测试
  - 情景实验室：用自然语言描述假设，自动生成冲击规则并计算影响

### 3.8 告警中心

- 支持三种告警类型：
  - **价格告警**（大于等于 / 小于等于目标价）
  - **事件告警**（特定事件类型触发）
  - **研报告警**（新研报 / 业绩预告）
- 告警中心视图展示规则列表与触发状态
- 实时计算规则与当前市场数据的偏离

### 3.9 策略与回测

- 策略评分与信号解释
- 回测结果展示（收益曲线、胜率、置信度）
- Backtest Confidence 指标
- 回测磁盘缓存，避免重复计算

### 3.10 管理后台（Admin）

仅管理员账号可见，入口在顶部导航栏。

**系统状态**：
- 应用名称、数据库连接、Redis 连接
- 内存缓存条目数与上限
- Redis 命中次数 / 未命中次数

**性能监控**（基于实时访问日志）：
- 平均响应时间
- P95 / P99 响应时间
- 5xx 错误率
- 实时 QPS
- 总请求数

**访问分析可视化**：
- 每小时访问量趋势折线图（ECharts 面积图）
- HTTP 状态码分布环形饼图
- Top 10 访问 IP 排行榜
- Top 10 访问路径排行榜
- 最近访问记录明细表（含 IP 地址、方法、路径、状态码、耗时）
数据每 10 秒自动刷新。

**缓存清理**：
- 一键清理全部缓存（内存 + Redis）
- 按前缀 / 通配符清理（如 `macro:*`）

**用户管理**：
- 分页查看所有注册用户
- 禁用 / 启用用户账号
- 分配 / 取消管理员权限

## 4. 认证与权限

- 用户可自由注册普通账号
- 管理员账号由环境变量 `AUTH_ADMIN_ACCOUNT` / `AUTH_ADMIN_PASSWORD` 指定
- 默认管理员账号：`admin` / `admin`
- 管理员身份通过数据库 `is_admin` 字段持久化，并在 JWT Token 中携带
- 后端提供 `require_admin` 统一鉴权中间件

## 5. 部署与运维

### 5.1 Docker 部署

项目已完整容器化，支持 `docker compose up --build -d` 一键启动：

```bash
# 启动 PostgreSQL + Redis + API + Web
docker compose up --build -d

# 访问前端 http://localhost:3000
# 访问 API 文档 http://localhost:8000/docs
```

### 5.2 CI/CD 自动发布

- 每次 Push 自动运行测试和构建检查
- Push 到 `master` 自动构建镜像并推送到 GitHub Container Registry
- 自动 SSH 登录生产服务器并重启服务

### 5.3 ETL 自动化

服务器配置 `crontab` 即可实现每日自动数据抓取：

```cron
0 2 * * * cd ~/quantpulse && bash scripts/run-etl.sh
```
