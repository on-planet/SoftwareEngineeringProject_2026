# Vercel + Render Deployment

This project deploys as two services:

- Frontend: Vercel, root directory `repo/web`
- Backend: Render, Docker web service from `render.yaml`

## 1. Backend on Render

1. Push this repository to GitHub.
2. In Render, create a new Blueprint from the GitHub repository.
3. Render will read `render.yaml` and create `quantpulse-api`.
4. Fill these environment variables before the first deploy:

```text
DATABASE_URL=postgresql+psycopg2://USER:PASSWORD@HOST:PORT/DATABASE
REDIS_URL=redis://HOST:PORT/0
AUTH_ADMIN_PASSWORD=change-this-password
```

`AUTH_TOKEN_SECRET` is generated by Render from `render.yaml`.

If using Neon for PostgreSQL, the connection string can usually be used as
`postgresql://...`; SQLAlchemy will use psycopg2 automatically.

After creating the database, apply the schema:

```bash
psql "$DATABASE_URL" -f repo/db/init.sql
```

Then apply any required SQL files under `repo/db/migrations/` in filename order.

Health checks:

```text
/health/live
/health/ready
```

## 2. Frontend on Vercel

1. Import the same GitHub repository in Vercel.
2. Set Root Directory to:

```text
repo/web
```

3. Use the Next.js framework preset.
4. Set these Vercel environment variables:

```text
API_PROXY_TARGET=https://YOUR_RENDER_SERVICE.onrender.com
NEXT_PUBLIC_API_PROXY_TARGET=https://YOUR_RENDER_SERVICE.onrender.com
```

5. Deploy.

The frontend calls local `/api/*` routes. `next.config.js` rewrites those
requests to the Render backend, so the browser keeps same-origin requests.
