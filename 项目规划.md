## 2. ETL 层已落地的核心模块

### 2.1 客户端封装

- `akshare_client.py` — 港股行情、宏观序列
- `baostock_client.py` — A 股基础数据
- `pysnowball_client.py` — 雪球行情、资讯
- `hkex_client.py` — 港交所回购披露文件

### 2.2 加载与落库

- `pg_loader.py`
  - `upsert_indices(rows)`
  - `upsert_daily_prices(rows)`
  - `upsert_financials(rows)`
  - `upsert_news(rows)`
  - `upsert_events(rows)`
  - ...

### 2.3 作业调度

- `index_job.py` — `run_index_job(date)`
- `financial_job.py` — `run_financial_job(date)`
- `news_job.py` — `run_news_job(date)`
- `macro_job.py` — `run_macro_job(date)`
- `futures_job.py` — `run_futures_job(date)`
- `sector_exposure_job.py` — `run_sector_exposure_job(date)`

### 2.4 调度器

- `scheduler.py` — 带锁文件（`repo/state/etl.lock`）的周期调度器
- 支持并行 worker
- 支持增量运行与断点恢复
- 支持保留期清理（7 天 / 30 天）

### 2.5 入口

- `main.py` — 单次运行入口，读取 `etl/config/settings.yml`
- `start_etl.cmd` / `stop_etl.cmd` — Windows 本地启停脚本
- `scripts/run-etl.sh` — 服务器自动化脚本（配合 `crontab`）
- Docker：`docker compose --profile etl run --rm etl`

## 3. API 层已落地的核心模块

### 3.1 Core

- `core/config.py` — 读取 `.env.local` 和 `settings.yml`
- `core/db.py` — PostgreSQL 连接池（SQLAlchemy）
- `core/cache.py` — Redis + 内存双层缓存，带统计接口
- `core/middleware.py` — 请求日志中间件，记录客户端 IP、耗时、状态码，维护内存访问日志环形缓冲区
- `core/security.py` — 自研 JWT（HMAC-SHA256），Payload 含 `is_admin`

### 3.2 Services

- `auth_service.py` — 注册、登录、管理员自动初始化
- `admin_service.py` — 用户管理、系统状态、缓存清理
- `index_service.py` — 指数列表与详情
- `stock_service.py` — 个股画像、日线、对比
- `macro_service.py` — 宏观快照与序列
- `news_service.py` — 新闻列表与关系图
- `portfolio_analysis_service.py` — 组合诊断
- `portfolio_stress_service.py` — 组合压力测试
- `strategy_service.py` — 策略评分与回测

### 3.3 Routers

- `auth.py` — `/auth/register`, `/auth/login`, `/auth/me`
- `admin.py` — 管理员接口：用户管理、系统状态、缓存清理、访问分析
- `user.py` — 用户投资组合 CRUD
- `user_targets.py` — 自选/已买标的
- `user_alerts.py` — 告警规则与告警中心
- `user_workspace.py` — 股票池与筛选器
- `index.py`, `stock.py`, `macro.py`, `news.py`, `events.py`, `strategy.py` — 对应业务域

路由通过 `routers/__init__.py` 自动扫描加载。

## 5. 跨层依赖关系

```
ETL (fetchers + transformers + loaders)
  -> PostgreSQL / Redis
      -> API (routers -> services -> models)
          -> Web (pages -> components -> services/api)
```

禁止反向依赖：
- API 不直接依赖 ETL 的实现细节
- 前端不直接访问数据库

## 6. 管理员权限模型（已落地）

1. 数据库 `auth_users.is_admin` 布尔字段标识管理员身份
2. 应用启动时 `ensure_admin_user()` 自动创建/同步管理员账号
3. JWT Token Payload 包含 `is_admin`
4. `require_admin` FastAPI 依赖提供后端鉴权
5. 前端 `AuthProvider` 通过 `user.is_admin` 控制管理员入口显隐
6. 管理后台页面 (`/admin`) 提供：
   - 系统状态监控
   - 性能监控（P95/P99、QPS、错误率）
   - 访问分析可视化（ECharts 折线图、饼图、IP/路径 Top 10）
   - 缓存清理
   - 用户管理（禁用/启用、分配管理员）
