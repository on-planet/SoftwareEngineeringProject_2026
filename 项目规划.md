## 2. ETL 层核心模块与函数

### 2.1 客户端封装
- `tushare_client.py`
  - `get_index_daily(date)`：获取 A 股指数日线。
  - `get_daily_prices(symbols, date)`：获取日线行情。
  - `get_financials(symbol, period)`：获取财报数据。
- `akshare_client.py`
  - `get_hk_daily(date)`：港股日线。
  - `get_macro_series(code, start, end)`：宏观序列。
- `hkex_client.py`
  - `download_buyback_file(date)`：下载港股回购披露。

### 2.2 转换与指标计算
- `indicators.py`
  - `calc_ma(series, window)`：均线。
  - `calc_rsi(series, window=14)`：RSI。
  - `calc_volatility(returns)`：波动率。
  - `calc_max_drawdown(series)`：最大回撤。
- `fundamentals.py`
  - `calc_profit_quality(net_income, cashflow)`。
  - `calc_growth(revenue, last_revenue)`。
  - `calc_risk(debt_ratio, short_debt, cash)`。
  - `calc_fundamental_score(...)`：汇总评分。
- `macro.py`
  - `normalize_macro_value(series)`：标准化宏观指标。

### 2.3 加载与落库
- `pg_loader.py`
  - `upsert_indices(rows)`
  - `upsert_daily_prices(rows)`
  - `upsert_financials(rows)`
  - `upsert_fundamental_score(rows)`
  - `upsert_news(rows)`
  - `upsert_events(rows)`
- `redis_cache.py`
  - `cache_heatmap(date, data)`
  - `cache_risk(symbol, data)`

### 2.4 作业调度
- `index_job.py`
  - `run_index_job(date)`：拉取指数 -> 落库。
- `financial_job.py`
  - `run_financial_job(date)`：财报 -> 评分 -> 写库。
- `news_job.py`
  - `run_news_job(date)`：新闻抓取 -> 情绪 -> 写库。
- `macro_job.py`
  - `run_macro_job(date)`：宏观数据落库。

### 2.5 入口与调度器
- `main.py`
  - `main()`：读取配置，按日期触发各 job（支持 T‑1）。

# 🔗 结构之间的依赖关系

> 以“调用方向”为准：上层依赖下层，避免反向依赖。

## 1. ETL 依赖
- `etl/main.py` 依赖 `etl/jobs/*`。
- `etl/jobs/*` 依赖 `fetchers/*` + `transformers/*` + `loaders/*`。
- `transformers/*` 仅依赖 `utils/*`，不依赖 DB。
- `loaders/*` 依赖 `pg_loader.py` + `redis_cache.py`。

## 2. API 依赖
- `routers/*` 依赖 `services/*`。
- `services/*` 依赖 `models/*` + `core/db.py` + `core/cache.py`。
- `models/*` 仅依赖 ORM/数据库配置，不依赖 `routers/*`。
- `schemas/*` 依赖 `models/*`（用于序列化/响应）。

## 3. 前端依赖
- `pages/*` 依赖 `components/*` + `services/api.ts`。
- `components/*` 依赖 `services/api.ts` + `utils/format.ts`。
- `services/api.ts` 依赖 API 路由契约（与后端 `routers/*` 对齐）。

## 4. 跨层依赖（数据流）
- `ETL -> PostgreSQL/Redis -> API -> 前端`。
- 禁止：`API` 反向依赖 `ETL`；`前端` 直接访问数据库。

## 5. 典型调用链示例
- `web/pages/index.tsx` → `services/api.ts:getIndices()` → `api/routers/index.py` → `api/services/index_service.py` → `api/models/indices.py` → `PostgreSQL`。
- `etl/jobs/financial_job.py` → `fetchers/tushare_client.py` + `transformers/fundamentals.py` → `loaders/pg_loader.py` → `PostgreSQL`。
